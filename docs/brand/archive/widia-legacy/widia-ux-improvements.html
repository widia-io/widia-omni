02

Eliminar ambiguidade: botão + input

Ter um botão "+ Nova tarefa" E um input aberto abaixo confunde. Uma única affordance.

❌ Antes — dois pontos de entrada
+ Nova tarefa

Usuário não sabe se clica no botão ou digita no input. Dois caminhos para a mesma ação = hesitação.

✓ Depois — input único com ação inline
+
Criar

Input inline fixo. Botão "Criar" aparece habilitado só quando há texto. Atalho: Enter para criar.

💻 Implementação

Remover o botão separado. O input inline é sempre visível. Botão "Criar" com opacity: 0.4 quando vazio, opacity: 1 quando tem texto. Submeter com Enter ou clique.

const [task, setTask] = useState('');

<div className="flex items-center gap-3">
  <Plus size={18} className="text-accent" />
  <input
    value={task}
    onChange={e => setTask(e.target.value)}
    onKeyDown={e => e.key === 'Enter' && task && handleCreate()}
    placeholder="Adicionar tarefa..."
  />
  <button
    onClick={handleCreate}
    disabled={!task}
    className={task ? 'opacity-100' : 'opacity-40 pointer-events-none'}
  >Criar</button>
</div>

Limpar filtros

Tarefa criada

1) Input limpa e refoca. 2) Tarefa entra com slide + highlight sutil. 3) Toast "Tarefa criada" por 2s.

💻 Fluxo de criação

async function handleCreate() {
  const newTask = await createTask({ name: task, priority, area });

  // 1. Limpa input e refoca
  setTask('');
  inputRef.current?.focus();

  // 2. Adiciona com flag de animação
  setTasks(prev => [{ ...newTask, isNew: true }, ...prev]);

  // 3. Toast
  toast.success('Tarefa criada', { duration: 2000 });

  // 4. Remove highlight após animação
  setTimeout(() => {
    setTasks(prev => prev.map(t =>
      t.id === newTask.id ? { ...t, isNew: false } : t
    ));
  }, 1500);
}

// Na task row:
<div className={cn(
  'task-row',
  task.isNew && 'animate-slide-in bg-accent/5 border border-accent/10'
)} />
07

Micro-copy contextual

Placeholders e labels genéricos não guiam. Texto contextual reduz hesitação e acelera ação.

Mapeamento de micro-copy

Elemento Antes Depois
Placeholder do input "Nome da tarefa" "O que precisa ser feito hoje?"
Botão de criar "Criar" "✓ Criar" (ou só Enter ↵)
Lista vazia (nada) "🎯 Que tal planejar algo importante?"
Filtro ativo "Status" "Abertas (3)"
Toast de confirmação (nenhum) "✓ Tarefa criada"
Concluir tarefa (nenhum) "Feito! 🎉" (toast 1.5s)
Seção de concluídas (nenhum) "▸ 3 tarefas concluídas"

💡 Princípio

Cada texto no sistema deve responder "o que eu faço aqui?" ou "o que acabou de acontecer?". Placeholder guia a ação, labels mostram estado, toasts confirmam resultado. Se o texto não faz nenhuma dessas 3 coisas, ele é ruído.
