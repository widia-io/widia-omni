03 — Implementação
CSS Design Tokens

Use prefers-color-scheme ou uma classe .dark no body. Os tokens são drop-in — basta trocar as variáveis.

☀️ Light Mode — :root 
:root {
  /* Background layers */
  --bg: #FDFCFA;
  --surface: #F6F4F0;
  --card: #FFFFFF;
  --border: #E5E2DC;

  /* Text */
  --text: #1A1A2E;
  --text-secondary: #6B7280;
  --text-muted: #9CA3AF;

  /* Accent */
  --accent: #E8580C;
  --accent-hover: #D14D08;
  --accent-light: #FFF3ED;

  /* Semantic */
  --success: #16A34A;
  --info: #6366F1;
  --warning: #EAB308;
  --danger: #DC2626;

  /* Areas */
  --area-saude: #EF4444;
  --area-carreira: #6366F1;
  --area-financas: #16A34A;
  --area-relac: #EC4899;
  --area-dev: #F59E0B;

}

04 — Dicas
Implementação do Dual Theme

🎯 Princípio-chave

No dark mode, o acento laranja sobe de luminosidade (#E8580C → #F0874A) para manter o mesmo contraste perceptual. O mesmo vale para todas as cores semânticas — suba ~2 stops na escala Tailwind (ex: red-500 → red-400).

🏗️ Estrutura CSS

Use var(--bg), var(--surface), var(--card) como 3 camadas. Nunca hardcode cores. No dark, a hierarquia se inverte: fundos ficam mais claros conforme sobem na camada. Mantenha bordas sempre com var(--border).

🔄 Toggle de tema

Use class="dark" no <html> + localStorage para persistir. Respeite prefers-color-scheme como default. Em Next.js/React, use next-themes ou um context simples com useEffect.

🌗 Backgrounds com textura

No dark mode, evite preto puro (#000). O #0F0F17 tem um toque de azul-roxo que combina com o warm do light. Isso mantém a personalidade "sand" sem ficar genérico como cinza puro.

📐 Opacidade como aliada

Para chip de área, use background: rgba(var(--area-cor), 0.06) no light e 0.10 no dark. Mesma regra para hover states. Isso funciona automaticamente com qualquer cor de área.

🖼️ Sombras vs Bordas

No light, sombras funcionam bem para elevar cards. No dark, sombras somem visualmente — use bordas sutis (var(--border)) como separador principal. Combine os dois para o efeito ideal.
